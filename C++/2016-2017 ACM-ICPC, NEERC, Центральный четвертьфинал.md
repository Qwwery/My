## **A. Жареная рыба**

Известно, что Рыбинск когда-то был отличным местом для рыбалки. Жители ели рыбу на завтрак, обед и ужин, а излишки отправляли в другие города. Поэтому оптимизация процесса жарки рыбы была важной задачей.

Сегодня поймано **N** рыб одинакового размера. На сковороду помещается не более **K** рыб одновременно. Каждую сторону каждой рыбы нужно жарить ровно **1 минуту** (то есть каждую рыбу — 2 минуты в сумме, по минуте на сторону).

Определите **минимальное время**, необходимое для жарки всех рыб.

**Ограничения:**  
1 ≤ N, K ≤ 500

**Вход:** два целых числа N и K  
**Выход:** одно целое число — минимальное время в минутах

## **B. Ханойская башня**

Есть 3 стержня: A, B, C. Изначально на стержне A лежат **N** дисков разного диаметра, упорядоченных по возрастанию снизу вверх (самый маленький — сверху). Стержни B и C пусты.

Цель — перенести все диски с A на B, используя C как вспомогательный, соблюдая правило: нельзя класть больший диск на меньший.

Вам нужно найти **номер хода**, после которого впервые количество дисков на всех трёх стержнях станет **одинаковым** (то есть N делится на 3, и на каждом стержне по N/3 дисков).

**Ограничения:**  
1 ≤ N ≤ 300, N делится на 3

**Вход:** N  
**Выход:** номер хода

## **C. Рабочий стол**

Вася хочет разместить как можно больше ярлыков на рабочем столе размером **h × w** клеток. Каждый ярлык — квадрат **2 × 2**, его углы должны совпадать с границами клеток. Ярлыки могут **перекрываться**, но **как минимум 2 клетки** (половина) каждого ярлыка должны быть видны.

Нужно найти **максимальное количество ярлыков** и вывести координаты **верхнего левого угла** каждого (нумерация строк и столбцов с 1).

**Ограничения:**  
1 ≤ h, w ≤ 500

**Вход:** h w  
**Выход:** сначала число N, затем N строк с координатами

## **D. Метеостанция**

Метеостанция записывает направления ветра без разделителей. Направления кодируются символами **N, S, E, W**, но некоторые направления — **двухсимвольные**: NE, NW, SE, SW, EN, WN и т.д. (всего 8 направлений, каждое — либо 1, либо 2 символа).

Дана строка из символов N/S/E/W. Требуется найти **количество способов разбить её на корректные направления**. Ответ вывести по модулю **10⁹ + 7**.

**Ограничения:** длина строки ≤ 10⁵

**Пример:**  
`NEWS` → `{N,E,W,S}` и `{NE,W,S}` → ответ 2

## **E. Кексы**

В очереди **N** студентов. На столе **K** кексов. Каждый студент **i** может съесть от 1 до **aᵢ** кексов. После хода он уходит в конец очереди. Процесс продолжается, пока кексы не кончатся. Тот, кто подходит к пустому столу — **убирает**.

Среди студентов есть **Жадина** — у него **aᵢ строго больше всех остальных**. Он всегда ест **min(aᵢ, остаток кексов)**.

Все остальные хотят, чтобы **Жадина убирал**. Возможно ли это при правильном выборе количества кексов, съедаемых каждым (в пределах их лимитов)?

**Вывести:** `YES`, если возможно, иначе `KEK`.

## **F. Витамины**

Есть **n** таблеток, каждая — **белая (W)**, **красная (R)** или **синяя (B)**. Их веса: W > R > B. Даны **m** сравнений вида `i < j`, `i > j`, `i = j`.

Нужно определить цвет каждой таблетки, если возможно. Если цвет неоднозначен — вывести `?`.

**Вывод:** строка из n символов: W/R/B/?

## **G. Сфенические числа**

Сфеническое число — это произведение **ровно трёх различных простых чисел**.

Дано число **n**. Вывести `YES`, если оно сфеническое, иначе `NO`.

**Примеры:**  
30 = 2·3·5 → YES  
40 = 2³·5 → NO

---

## **H. Неслучайные числа**

Генератор выдаёт **n-значные** числа без ведущих нулей, но с ограничением: **на i-й позиции (слева)** не может стоять цифра **i**.

Например, при n=2: нельзя, чтобы первая цифра была 1, а вторая — 2.

Сколько таких чисел существует?

**Ограничения:** 1 ≤ n ≤ 100

**Вывод:** количество (без ведущих нулей)

## **I. Раздел земли**

Дан выпуклый **N-угольник** (координаты по часовой стрелке). Нужно провести **прямой забор** между двумя точками на границе так, чтобы получились два многоугольника: один с **K** сторонами, другой — с **M** сторонами.

Найти **минимальную длину забора** с точностью до 3 знаков после запятой. Если невозможно — вывести `-1`.

**Ограничения:** 2 < N, K, M < 100

## **J. Архитектор своей удачи**

Есть **n** автобусных и **m** трамвайных билетов (шестизначные номера). Из двух билетов можно сделать **"суперсчастливый"**, если:

- взять **первые 3 цифры** одного билета и **последние 3** другого,
- и сумма первых трёх = сумме последних трёх.

Каждый билет можно использовать **только один раз**.

Нужно максимизировать число таких пар и вывести их. Указывать, откуда взята левая часть:  
- `AT` — автобус + трамвай  
- `TA` — трамвай + автобус

## **K. Полиморфный код**

Дан "зашифрованный" ассемблероподобный код, полученный из исходного путём замены команд на эквивалентные последовательности (см. таблицу замен в условии).

Требуется **восстановить исходный код минимальной длины**, эквивалентный данному.

Команды: MOV, ADD, XOR, PUSH, POP с регистрами R1–R8 и константами (в hex).

**Вход:** n команд  
**Выход:** k — число команд в восстановленном коде, затем сами команды
