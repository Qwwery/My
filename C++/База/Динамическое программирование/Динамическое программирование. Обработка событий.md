Событие (отрезок / интервал)

Несколько этапов:
1) Придумать способ хранения 
2) Сортировка 
3) Линейный пробег
Чаще всего храним в виде vector<pair<int, int>>;
Сортировка методом sort();
Линейный пробег циклом for()

## Задача:
Пусть у нас есть краска
$$
l_i, r_i
$$

```cpp
int n;
cin >> n;
vector<pair<int, int>> A;
int time, mark;
pair<int, int> tmp;
for (int i = 0; i < n; i++) {
	cin >> time;
	mark = -1;
	tmp.first = time;
	tmp.second = mark;
	A.push_back(tmp);

	cin >> time;
	mark = 1;
	tmp.first = time;
	tmp.second = mark;
	A.push_back(tmp);
}
sort(A.begin(), A.end());
int res = 0;
int count = 0;
int time_last = 0;
int time_cur = 0;
for (int i = 0; i < A.size(); i++) {
	time_last = time_cur;
	time_cur = A[i].first;
	if (count != 0)
		res += time_cur - time_last;
	if (A[i].second == -1)
		count++;
	else
		count--;
}
cout << res;
```

mark нужен (в vector<pair<int, int>>) для качественной сортировки в том случае, когда совпадает время закраски забора.

## Задача 2
Человек приходит в магазин в time_in, а уходит в time_out
Нужно запустить рекламу, так, чтобы её прослушало как можно больше людей
Реклама считается прослушанной тогда, когда её полностью прослушали 

Реализация:
T - время рекламы
меняем [time_in_1, time_out_2] на [time_in_ш, time_out_i - T]
если time_out - T < time_in, то человек физически не может прослушать рекламу
Находим, в какое промежуток времени в магазине наибольшее количество людей

## Задача 3
Есть задачи цикличные (например, школа открывается в 7)

Этапы решения:
Преобразуем данный цикл в прямую (разрываем цикл). если событие начинается до полуночи, а заканчивается после, то разрываем его так, чтобы оно заканчивалось в 0:00 и начиналось на следующий день (максимальное увеличение событий в 2 раза)
